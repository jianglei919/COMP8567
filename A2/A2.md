# A2 功能文档

本文档按当前 `A2_Lei_Jiang.c` 的**实际实现**整理，分为中英文两个版本。

- 可执行程序：`./A2`
- 基本用法：
    - `./A2 root_process process_id [option]`
    - `./A2 -bcp`
    - `./A2 -bop`

---

## 中文版（Chinese）

### 1) 默认功能

#### 无 option（`./A2 root_process process_id`）
- 功能描述：判断 `process_id` 是否属于以 `root_process` 为根的子树。
- 实现逻辑：
    1. 从 `process_id` 沿 PPid 向上追父链；
    2. 若追溯过程中遇到 `root_process`，输出 `process_id root_process`；
    3. 否则输出不属于该子树的信息并返回失败。
- 测试方法：
    1. 启动 `a2sampletree`，记录主进程 PID 作为 `root_process`；
    2. 选一个后代 PID：`./A2 <root> <descendant>`，应输出两列 PID；
    3. 选一个无关 PID：`./A2 <root> <other_pid>`，应提示不在子树中。

### 2) 统计/查询类

#### `-cnt`
- 功能描述：统计 `process_id` 的后代总数。
- 实现逻辑：调用 `collect_descendants(process_id, ...)` 收集后代并输出数量。
- 测试方法：`./A2 <root> <pid> -cnt`，与手工 `ps --forest` 对照数量。

#### `-oct`
- 功能描述：统计“当前后代集合”中的孤儿数量。
- 实现逻辑：
    1. 先收集当前仍可追溯到 `process_id` 的后代集合；
    2. 对每个后代，若其父进程为 `1`，或父进程不在该后代集合中（且非直接子进程），计为孤儿。
- 测试方法：`./A2 <root> <pid> -oct`，结合 `ps -o pid,ppid,stat,cmd` 检查 PPID=1 及父节点缺失情况。

#### `-odt`
- 功能描述：查询最早创建的后代 PID，并输出可读创建时间。
- 实现逻辑：
    1. 扫描后代，选 `start_ticks` 最小者（并列取 PID 更小）；
    2. 用 `/proc/stat` 的 `btime` + `start_ticks/CLK_TCK` 转换时间；
    3. 按格式输出。
- 测试方法：`./A2 <root> <pid> -odt`，与 `ps -o lstart -p <pid>` 交叉验证。

#### `-ndt`
- 功能描述：查询最新创建的后代 PID。
- 实现逻辑：扫描后代，选 `start_ticks` 最大者（并列取 PID 更小）。
- 测试方法：`./A2 <root> <pid> -ndt`，新建一个后代进程后应输出该新 PID 或同级最新 PID。

#### `-dnd`
- 功能描述：统计非直接后代数量。
- 实现逻辑：`non_direct = descendants_total - direct_children_count`。
- 测试方法：`./A2 <root> <pid> -dnd`，结合树结构手工核对“孙子及更深层”数量。

#### `-mmd`
- 功能描述：查询内存占用（VmRSS）最大的后代，支持并列输出。
- 实现逻辑：
    1. 遍历后代取最大 `vmrss_bytes`；
    2. 再遍历输出所有等于最大值的 PID；
    3. 输出 `VmRSS: <bytes>`。
- 测试方法：`./A2 <root> <pid> -mmd`，与 `cat /proc/<pid>/status | grep VmRSS` 对照。

#### `-mpd`
- 功能描述：查询累计 CPU 时间（`utime+stime`）最大的后代，支持并列输出。
- 实现逻辑：
    1. 对每个后代计算 `cpu_ticks = utime_ticks + stime_ticks`；
    2. 用 64 位无符号整数找最大值并输出并列 PID；
    3. 输出 `Total CPU time: <ticks> clock ticks`。
- 测试方法：`./A2 <root> <pid> -mpd`，与 `cat /proc/<pid>/stat` 的字段 14+15 对照。

### 3) 控制/信号类

#### `-sst`
- 功能描述：向 `process_id` 的兄弟进程发送 `SIGSTOP`。
- 实现逻辑：
    1. 用 `collect_siblings` 找同一父进程的兄弟；
    2. 跳过 bash；
    3. 对其余兄弟发送 `SIGSTOP`。
- 测试方法：`./A2 <root> <pid> -sst`，再用 `ps -o pid,stat -p <sibling_pid>` 检查状态是否为 `T`。

#### `-sco`
- 功能描述：向已停止的兄弟进程发送 `SIGCONT`。
- 实现逻辑：
    1. 收集兄弟进程；
    2. 仅对状态 `T` 的兄弟发送 `SIGCONT`；
    3. 跳过 bash。
- 测试方法：先执行 `-sst`，再执行 `./A2 <root> <pid> -sco`，观察状态从 `T` 恢复。

### 4) kill 类

#### `-dtm`
- 功能描述：按创建时间从晚到早，向后代发送 `SIGKILL`。
- 实现逻辑：后代按 `start_ticks` 降序排序后逐个 `kill(SIGKILL)`，跳过 bash。
- 测试方法：`./A2 <root> <pid> -dtm`，之后用 `ps` 验证后代被终止顺序与结果。

#### `-kpp`
- 功能描述：终止 `process_id` 的父进程。
- 实现逻辑：读取父 PID，若不是系统关键进程且非 bash，则 `SIGKILL`。
- 测试方法：`./A2 <root> <pid> -kpp`，用 `ps -p <parent_pid>` 验证。

#### `-kgp`
- 功能描述：终止 `process_id` 的祖父进程。
- 实现逻辑：读取父 PID 和祖父 PID；若祖父合法且非 bash，则 `SIGKILL`。
- 测试方法：`./A2 <root> <pid> -kgp`，验证祖父进程退出。

#### `-ksp`
- 功能描述：终止 `process_id` 的所有兄弟进程。
- 实现逻辑：收集兄弟 PID，跳过 bash，逐个 `SIGKILL`。
- 测试方法：`./A2 <root> <pid> -ksp`，检查兄弟 PID 是否消失。

#### `-kps`
- 功能描述：终止 `process_id` 父进程的兄弟进程。
- 实现逻辑：先找父 PID，再收集父进程的兄弟，跳过 bash 后逐个 `SIGKILL`。
- 测试方法：`./A2 <root> <pid> -kps`，验证“叔伯”进程是否终止。

#### `-kcp`
- 功能描述：终止 `process_id` 的直接子进程。
- 实现逻辑：收集后代后筛选 `ppid == process_id`，逐个 `SIGKILL`（跳过 bash）。
- 测试方法：`./A2 <root> <pid> -kcp`，检查直接孩子是否退出。

#### `-kgc`
- 功能描述：终止 `process_id` 的孙子进程。
- 实现逻辑：
    1. 先识别直接子进程；
    2. 再筛选“父在直接子进程列表中”的进程并 `SIGKILL`。
- 测试方法：`./A2 <root> <pid> -kgc`，确认孙子层进程退出。

#### `-krp`
- 功能描述：终止 `root_process`。
- 实现逻辑：若 `root_process` 合法且非 bash，则发送 `SIGKILL`。
- 测试方法：`./A2 <root> <pid> -krp`，验证 root 进程是否终止。

### 5) 其他类

#### `-bcp`（无 root/process 参数）
- 功能描述：统计当前 bash 子树中的进程总数（不含 bash 本身）。
- 实现逻辑：找到当前 bash 祖先进程，再统计其后代数量。
- 测试方法：`./A2 -bcp`，与当前终端 `pstree -p` 对照。

#### `-bop`（无 root/process 参数）
- 功能描述：统计所有 bash 终端子树中的进程总数（去重，不含 bash 本身）。
- 实现逻辑：
    1. 扫描所有 bash PID；
    2. 汇总每个 bash 的后代 PID；
    3. 去重后输出总数。
- 测试方法：`./A2 -bop`，与多终端 `ps` 汇总结果进行近似对照。

#### 参数校验与错误处理
- 功能描述：统一入口参数合法性检查与选项分发。
- 实现逻辑：
    1. 校验 `root_process/process_id` 必须是正整数；
    2. 未知选项返回失败码；
    3. 各功能内部失败时打印错误并返回。
- 测试方法：
    - `./A2 abc 123`（非法参数）
    - `./A2 1 2 -unknown`（未知选项）

## English Version

### 1) Default Function

#### No option (`./A2 root_process process_id`)
- Description: Check whether `process_id` belongs to the subtree rooted at `root_process`.
- Implementation Logic:
    1. Walk up the PPid chain from `process_id`.
    2. If `root_process` is reached, print `process_id root_process`.
    3. Otherwise, print the “not in subtree” message and return failure.
- Test Method:
    1. Start `a2sampletree` and record its main PID as `root_process`.
    2. Run `./A2 <root> <descendant>` and expect a two-PID output.
    3. Run `./A2 <root> <unrelated_pid>` and expect a rejection message.

### 2) Statistics / Query

#### `-cnt`
- Description: Count total descendants of `process_id`.
- Implementation Logic: Use `collect_descendants(process_id, ...)` and print the count.
- Test Method: `./A2 <root> <pid> -cnt`; compare with manual tree count.

#### `-oct`
- Description: Count orphan processes within the current descendant set.
- Implementation Logic:
    1. Collect descendants currently traceable to `process_id`.
    2. For each descendant, if parent is `1` or parent is outside this set (excluding direct children), count as orphan.
- Test Method: `./A2 <root> <pid> -oct`; verify with `ps -o pid,ppid,stat,cmd`.

#### `-odt`
- Description: Find the oldest descendant and print PID + readable creation time.
- Implementation Logic:
    1. Pick minimum `start_ticks` (tie-break by smaller PID).
    2. Convert using `/proc/stat` `btime` + `start_ticks/CLK_TCK`.
- Test Method: `./A2 <root> <pid> -odt`; compare with `ps -o lstart -p <pid>`.

#### `-ndt`
- Description: Find the newest descendant PID.
- Implementation Logic: Pick maximum `start_ticks` (tie-break by smaller PID).
- Test Method: `./A2 <root> <pid> -ndt`; create a new child and re-check.

#### `-dnd`
- Description: Count non-direct descendants.
- Implementation Logic: `non_direct = total_descendants - direct_children`.
- Test Method: `./A2 <root> <pid> -dnd`; verify using subtree levels.

#### `-mmd`
- Description: Find descendant(s) with maximum VmRSS.
- Implementation Logic:
    1. Scan descendants for max `vmrss_bytes`.
    2. Print all descendants tied at max.
    3. Print final VmRSS in bytes.
- Test Method: `./A2 <root> <pid> -mmd`; compare with `/proc/<pid>/status` VmRSS.

#### `-mpd`
- Description: Find descendant(s) with maximum cumulative CPU ticks.
- Implementation Logic:
    1. Compute `utime + stime` per descendant.
    2. Use 64-bit unsigned max tracking and print ties.
    3. Print total ticks.
- Test Method: `./A2 <root> <pid> -mpd`; compare with `/proc/<pid>/stat` fields 14+15.

### 3) Control / Signal

#### `-sst`
- Description: Send `SIGSTOP` to siblings of `process_id`.
- Implementation Logic: Collect siblings, skip bash, send `SIGSTOP`.
- Test Method: `./A2 <root> <pid> -sst`; check sibling state `T` via `ps`.

#### `-sco`
- Description: Send `SIGCONT` to stopped siblings.
- Implementation Logic: Collect siblings, filter state `T`, skip bash, send `SIGCONT`.
- Test Method: run `-sst` first, then `-sco`; verify state recovery.

### 4) Kill Category

#### `-dtm`
- Description: Kill descendants from newest to oldest.
- Implementation Logic: Sort by `start_ticks` descending, `SIGKILL` each (skip bash).
- Test Method: `./A2 <root> <pid> -dtm`; verify descendants are terminated.

#### `-kpp`
- Description: Kill parent process.
- Implementation Logic: Resolve parent PID, validate, `SIGKILL` unless bash/system-critical.
- Test Method: `./A2 <root> <pid> -kpp`; verify parent is gone.

#### `-kgp`
- Description: Kill grandparent process.
- Implementation Logic: Resolve parent then grandparent, validate, send `SIGKILL`.
- Test Method: `./A2 <root> <pid> -kgp`; verify grandparent exits.

#### `-ksp`
- Description: Kill all siblings of `process_id`.
- Implementation Logic: Collect siblings, skip bash, send `SIGKILL`.
- Test Method: `./A2 <root> <pid> -ksp`; verify sibling PIDs are terminated.

#### `-kps`
- Description: Kill siblings of parent process.
- Implementation Logic: Get parent PID, collect parent’s siblings, skip bash, kill all.
- Test Method: `./A2 <root> <pid> -kps`; verify those processes are terminated.

#### `-kcp`
- Description: Kill direct children of `process_id`.
- Implementation Logic: Filter descendants by `ppid == process_id`, skip bash, kill.
- Test Method: `./A2 <root> <pid> -kcp`; verify direct children are gone.

#### `-kgc`
- Description: Kill grandchildren of `process_id`.
- Implementation Logic: Identify direct children first, then kill processes whose parent is one of those children.
- Test Method: `./A2 <root> <pid> -kgc`; verify grandchild-level processes are terminated.

#### `-krp`
- Description: Kill `root_process`.
- Implementation Logic: Validate root PID and bash-protection, then `SIGKILL`.
- Test Method: `./A2 <root> <pid> -krp`; verify root process exits.

### 5) Others

#### `-bcp` (no root/process arguments)
- Description: Count processes in current bash subtree (excluding bash itself).
- Implementation Logic: Find current bash ancestor, collect descendants, print count.
- Test Method: `./A2 -bcp`; compare with current terminal process tree.

#### `-bop` (no root/process arguments)
- Description: Count processes across all bash terminal subtrees (deduplicated, excluding bash processes).
- Implementation Logic:
    1. Find all bash PIDs.
    2. Merge descendants from each bash.
    3. Deduplicate and print count.
- Test Method: `./A2 -bop`; compare against multi-terminal `ps` summary.

#### Input Validation / Dispatch
- Description: Validate input and dispatch options safely.
- Implementation Logic:
    1. `root_process` and `process_id` must be positive integers.
    2. Unknown options return failure.
    3. Per-option runtime failures print errors and return.
- Test Method:
    - `./A2 abc 123`
    - `./A2 1 2 -unknown`
